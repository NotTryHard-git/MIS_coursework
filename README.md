1. Архитектура и технологии
Приложение представляет собой одностраничное веб-приложение (SPA), реализованное на фреймворке Vue 3 с использованием Options API. В качестве источника данных и единого хранилища состояния выступает Pinia-стор (useStore), что обеспечивает реактивность данных между компонентами без прямого обмена событиями.
Основные технологии:
•	Vue 3 (Composition API / Options API)
•	Pinia (управление состоянием)
•	Vue Router (навигация)
•	Axios (HTTP-запросы к бэкенду)
•	Bootstrap Icons / FontAwesome (иконография)
Ключевые архитектурные решения:
1.	Модульность: Интерфейс разбит на переиспользуемые компоненты (SearchDetailPanel, LocationItem, ModalAddEquip и др.).
2.	Ролевая модель: Интерфейс адаптируется под права текущего пользователя (администратор видит кнопки добавления, редактирования, удаления; обычный пользователь — только просмотр и скачивание файлов).
3.	Централизованная логика: Все API-вызовы, управление загрузкой файлов, фильтрация и поиск сосредоточены в Pinia-сторе.

2. Структура и навигация приложения
Приложение состоит из трех основных разделов, доступных после аутентификации:
1.	Пользователи (доступен только администратору)
2.	Каталог оборудования
3.	Местоположение (структура складов, доступен только администратору)
Навигация реализована через компонент AdminNavPanel.vue, который содержит ссылки на соответствующие маршруты.
 
Верхняя панель Header.vue содержит логотип, глобальную строку поиска и информацию о текущем пользователе с кнопкой выхода.
3.3. Управление пользователями
Раздел реализован в компоненте AdminUsers.vue. Интерфейс отображает всех зарегистрированных пользователей в виде сетки карточек (UserItem.vue).
Функционал администратора:
•	Добавление нового пользователя через модальное окно (ModalAddUser.vue).
•	При добавлении задаются: ФИО, логин, пароль и флаг is_admin (права администратора).




3.4. Управление структурой складов
Раздел AdminLocation.vue отвечает за иерархию «Цех → Склад». Компонент LocationItem.vue отображает карточку цеха со списком принадлежащих ему складов.
Реализованные возможности:
•	Добавление нового цеха (метод addWorkshopHandler).
•	Добавление нового склада внутри существующего цеха (addWarehouseHandler).
•	Для каждого склада динамически подсчитывается количество единиц оборудования через store.getEquipmentByLocation(warehouse.id).
 

3.5. Каталог оборудования (основной функционал)
Раздел AdminCatalog.vue реализует основную бизнес-логику приложения.
3.5.1. Фильтрация по типам
Слева расположен сайдбар с категориями: насосы, электродвигатели, станки. При клике на категорию обновляется список оборудования и изменяется активное состояние. Данные подгружаются из store.typesList.
 
 3.5.2. Карточки оборудования
Каждое оборудование отображается компонентом SearchDetailPanel.vue. Карточка содержит:
•	Название и тип;
•	Местоположение (Цех № / Склад №);
•	Статус (На складе / в эксплуатации);
•	Динамические характеристики (в зависимости от типа);
•	Блок документов (Паспорт, Документация) с возможностью скачивания и (для админа) загрузки/удаления.
 

3.5.3. Поиск внутри категории
Пользователь может выполнить точный поиск по характеристикам, используя формат Параметр: Значение (например, Мощность двигателя (кВт): 200). Поиск отправляется в метод performSearch, который вызывает store.SearchInType.

3.5.4. Добавление/редактирование/удаление оборудования
Администратору доступна кнопка «Добавить», открывающая модальное окно ModalAddEqup.vue. Форма включает:
•	Выбор цеха и склада;
•	Выбор категории (подгружаются соответствующие поля характеристик);
•	Загрузку файлов (паспорт, документация);
•	Режим редактирования с возможностью удаления оборудования и замены файлов.
 
3.6. Глобальный поиск
Компонент SearchModal.vue обеспечивает быстрый поиск по названию оборудования во всем каталоге. Особенности реализации:
•	Результаты группируются по типам оборудования (groupedResults).
При выборе результата пользователь переходит на страницу детального просмотра.
 


3.7. Управление файлами
Приложение поддерживает загрузку и скачивание документов (PDF, DOC, изображения). Реализация включает:
•	Отслеживание состояния файлов (hasPassport, hasDocumentation).
•	Возможность удаления текущего файла с заменой на новый.
•	Отображение статуса «Загружен» / «Нет».
•	Загрузка через store.uploadFile с передачей FormData.
 

3.10. Система стилей
Единый файл main.css реализует Design System с CSS-переменными:
•	Тёмная промышленная тема (--c-bg, --c-surface, --c-border);
•	Акцентный цвет (--c-accent: #f59e0b);
•	Типографика (Roboto Condensed, Roboto Mono);
•	Переиспользуемые классы: карточки, кнопки, модальные окна, таблицы, бейджи, формы.
Такой подход обеспечивает единообразие интерфейса и упрощает поддержку.
4. Руководство пользователя (краткое)

Вход в систему	Любая	Ввести логин и пароль на странице авторизации

Заключение
Разработанное приложение «Складской каталог» представляет собой полноценную информационную систему для учета оборудования на складах. Реализован полный CRUD для пользователей, оборудования и локаций, а также файловое хранилище документов. Интерфейс отвечает современным требованиям (тёмная тема, адаптивная сетка).
